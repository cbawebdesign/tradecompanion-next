Zustand Store — addAlert() Dedup Logic

store.addAlert(alert)
  │
  ├─ Scan existing alerts[] for same symbol + type
  │   ├─ Exact message match? → SKIP (duplicate)
  │   └─ First 40 chars match (case-insensitive, min 10)? → SKIP (fuzzy dup)
  │
  ├─ Prepend to alerts[]
  ├─ Cap at 500 entries
  └─ Persist to localStorage

store.addAlerts(batch) (used by initial backfill)
  │
  ├─ Build Set of {symbol}|{message}|{type} from existing
  ├─ Filter batch to truly new (exact key only, no fuzzy)
  ├─ Prepend reversed batch (oldest first in new block)
  └─ Cap at 500 entries

Implementation Plan

Step 1: Web Alert Logger

Add a middleware layer in the Zustand store that intercepts every addAlert() call and POSTs a log entry to the Azure Function: POST /tcadmin/alert-log with {source: 'web', symbol, type, message, timestamp, hookName}. This requires zero UI changes — it's a store-level wrapper.

Step 2: Legacy Alert Logger

In AlertService.Alert_AlertTriggered(), add a fire-and-forget POST to the same endpoint: POST /tcadmin/alert-log with {source: 'legacy', symbol, type, message, timestamp, alertName}. One line of code.

Step 3: Comparison Endpoint

New Azure Function endpoint: GET /tcadmin/alert-diff?date=. Reads logged alerts from both sources, matches by symbol + type + 60-second time window, returns the diff report.

Step 4: Claude Computer Use (Performance Testing)

Run Claude Computer Use session during market hours pointed at the web version. Tasked with:

  • Monitoring alert bar for visual glitches (duplicates, flickering, frozen scroll)
  • Checking memory usage over time (chrome://process-internals or Performance tab)
  • Testing interactions: click symbol → verify AHK fires, click alert → verify URL opens, scroll → verify smooth
  • Screenshotting any anomaly for review
Step 5: Daily Automated Report

Timer trigger in Azure Function runs at market close (4:30pm ET). Calls the diff endpoint, formats a report, posts to a Slack/email/dashboard. Chris reviews the next morning.

Key advantage: This runs on LIVE production data. No mocking, no replay, no synthetic data. If Justin would have missed an alert, the system catches it. Runs unattended every market day.
