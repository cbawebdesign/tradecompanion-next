4. Agent Intake (human-fired triage)

What it is

Justin fires an issue into an admin form → agent assembles context across all 3 TC repos → produces a triage diagnosis → pings Chris → Chris reviews and walks through the fix.

Is not: autonomous PRs, chat interface for Justin, real-time debugging.

Data flow

Justin (tc-admin)
  → submits issue form
  → POST /api/tcadmin/issues → Cosmos TcIssues (status: pending)
  → Azure Function triggers triage (async)
      → CLAUDE.md files + git log + keyword code search
      → Claude API (Sonnet 4.6)
      → diagnosis JSON
  → Cosmos updated (status: triaged)
  → Notification to Chris
  → Chris reviews, walks Justin through or assigns to me

6. Alert Integrity Agent

What it is

Continuously verifies that alerts the backend received were also fired on the client. Generates a delivery score. Flags MISSED / DELAYED / DUPLICATE / MATCHED.

Current state

Partially built. AlertAuditTimer runs in the Azure Function and produces a delivery score. The score is broken — it reports ~11.9% because it counts "alert exists on backend but no client acknowledged" as a miss, even when no client was open to ack. Meaningless metric right now.

Fix (P1)

  1. Only compute delivery score for windows where at least one active client session existed (tracked via SignalR connection count).
  2. Split the metric: delivery when online vs recovered by auditor safety net vs never-delivered-and-never-recovered. The third is the one that matters.
  3. Surface per-alert-type breakdown (PR delivery vs Filing delivery vs Tweet delivery, etc).
  4. Write to Cosmos AlertIntegrityMetrics for tc-admin dashboard charts.

Future (v2)

  • Auto-create a TcIssues entry when a specific alert type's delivery drops > 10% below baseline (ties into Intake).
  • Detect deduplication regressions — same alert firing > N times in < M seconds across sessions.

7. Performance Monitor Agent

What it is

Tracks browser performance during a trading session: heap size, frame drops, long tasks (>50ms), DOM node count.

Scope

Can share infrastructure with Visual QA (both use headless Chromium). In practice: extend tc-qa-agent to emit a secondary performance report alongside the QA report on every run.

Metrics

  • Heap size via performance.measureUserAgentSpecificMemory() if available, fallback performance.memory.usedJSHeapSize
  • Long tasks via PerformanceObserver (type: 'longtask')
  • Frame drops via requestAnimationFrame sampling
  • DOM mutation rate via MutationObserver (already exposed by Playwright via CDP)

Baseline establishment

First N runs build a baseline ("steady state is 120MB heap, 800 DOM nodes"). Subsequent runs alert on deviations > 2× baseline.

Effort

~4–6 hours as an extension to tc-qa-agent. P3 — not blocking.

8. Report Aggregator

What it is

Daily summary, runs automatically at 4:30pm ET (after market close). Pulls from:

  • Alert Integrity Agent — delivery score + misses
  • Visual QA Agent — all Playwright runs from the day, aggregated pass/fail
  • Performance Monitor — memory/FPS trend for the session
  • Agent Intake — issues submitted, triaged, resolved

Output: a single daily report posted to tc-admin, with a notification to Chris if anything regressed vs previous day.

Scope

Requires Integrity metric fix (section 6) and Visual QA findings going to Cosmos (v0.3) before it has real data to aggregate. P2 — after Intake foundation + Integrity fix.

Effort

~4–6 hours. Mostly a Cosmos query + templating layer.

11. Risks & costs

Monthly cost estimate

Visual QA (tc-qa-agent)~$0 (GitHub Actions free tier)
Claude triage on failures~$5–20/mo at 10 issues/day
Intake triage calls~$60/mo at 10 submissions/day on Sonnet 4.6
Cosmos (new containers)~$0 at this volume
Blob storage (screenshots)~$1–3/mo
Total~$70–100/mo

Key risks

  • Bad agent diagnoses waste your time → frame everything as triage, never auto-fix
  • API key leaks via issue descriptions → add redaction pass
  • Agent hallucinates file paths → post-validate against GitHub search
  • Visual QA false positives → stable selectors (data-*), iterate thresholds
  • Notification fatigue → rate limit, severity thresholds, daily digest for low-urgency
