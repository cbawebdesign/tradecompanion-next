Legacy DesktopvsNext.js Web

Architecture deep-dive comparing the C#/React hybrid desktop app with the Next.js/Zustand web app. Same data, very different plumbing.

Legacy Desktop App C# + React

ASP.NET Core 6.0 server hosting a React 18 SPA. The C# backend manages all real-time connections, state, and alert logic. React is purely a rendering layer that talks to the local SignalR hub.

C# / .NET 6 ASP.NET Core React 18 TypeScript SignalR (3 hubs) Serilog File Config Cosmos DB Sync

Next.js Web App React + Zustand

Next.js 14 App Router with client-side Zustand state. All data fetching happens in the browser via hooks — SignalR for real-time, REST polling for alerts. Optional Electron wrapper for desktop.

Next.js 14 React 18 Zustand Tailwind CSS SignalR (2 hubs) Electron (optional) Vercel BroadcastChannel
⚙ Core Architecture
AspectLegacy DesktopNext.js Web
Runtime Model Server + Client — C# process runs on localhost:5000, React SPA on localhost:3000 (dev) or served from wwwroot (prod) Client-only — Everything runs in the browser. Next.js SSR used only for initial HTML shell. All data fetching is client-side.
State Management Static globals in C# — G.cs holds singletons: QuoteManager, AlertService, ConfigService. React uses component state only. Zustand store — Single client-side store with persist middleware (localStorage). Quotes, alerts, watchlists, config all in one place.
Config Storage File-based — ~/.config/LX/config.ini for settings, runningAlerts2.json for alerts. Synced to Cosmos DB by UserSyncService. localStorage — All config persisted via Zustand's persist middleware as 'trade-companion-storage' key.
Alert Logic Server-side (C#) — AlertService orchestrates all IAlert implementations. Alerts fire events, get routed through central service, then broadcast to React. Client-side (hooks) — Each alert type has its own hook (useFilingsPolling, useTweetsPolling, etc.). Each hook independently calls store.addAlert().
Entry Point Program.cs → G.cs (initializes all services) → index.tsx (React) app/layout.tsx → providers.tsx (wires hooks) → page.tsx (tabs)
Logging Serilog — Separate log files per feature: QuoteManager.txt, Alerts.txt, PRs.txt in ~/.config/LX/Logs/ console.log — Browser dev tools only. No persistent logging.
Deploy Target Local machine only — dotnet run on the user's desktop. Dead launcher at scanzzers.com/lx/v3build. Vercel (web) + optional Electron wrapper for desktop. Can run anywhere with a browser.
⚡ Real-Time Connections

Legacy — 3 SignalR Hubs

1. Azure Function Hub tradecompanion3.azurewebsites.net/api/ Quotes, TradingView, catalysts, filings, bars. Auth: negotiate endpoint.
2. News Hub (Direct) stage.news.scanzzers.com/newshub Press releases, filtered PRs. Auth: JWT via machine-login.
3. Local Hub localhost:5000/lxhub C# backend ↔ React frontend bridge. No auth (local only).

C# manages all 3 connections server-side. React only connects to local hub. Quotes are relayed: Azure → C# QuoteManager → local hub → React.

Next.js — 2 SignalR Hubs (Direct)

1. Azure Function Hub tradecompanion3.azurewebsites.net/api/ Quotes, alerts, scanner. Auth: negotiate. Connected directly from browser via useSignalR hook.
2. News Hub (Direct) stage.news.scanzzers.com/newshub Press releases. Auth: API key as bearer token. Connected via useNewsHub hook.
3. Local Hub — Not needed No C# middleman. Browser connects directly to Azure.

Browser connects directly to Azure SignalR + News hub. No local relay layer. Also uses 4 REST polling hooks for filings, tweets, catalysts, and trade exchange.

🗃 State Management
AspectLegacy DesktopNext.js Web
Pattern Static service singletons — G.QuoteManager, G.AlertService, etc. C# objects with in-memory dictionaries. Zustand global store — Single useStore() hook. Reactive selectors trigger re-renders.
Quote Storage QuoteManager._symbolL1Dict — C# Dictionary<string, L1>. Buffered 250ms, then broadcast to React as a flat object. store.quotes — Record<string, Quote>. Updated via updateQuotes(batch) every 250ms (hook-side batching).
Alert Storage AlertService._runningAlerts — C# list. File-persisted to runningAlerts2.json. React gets individual events via SignalR. store.alerts[] — JS array, max 500. Persisted to localStorage. Dedup by symbol|message|type.
Watchlists WatchlistManager — JSON files in ~/.config/LX/Watchlists/. Synced to Cosmos DB. store.watchlists[] — localStorage. No server sync (yet).
Persistence Files + Cosmos DB — Config.ini (local), JSON files (watchlists), Cosmos (cloud sync) localStorage only — Everything in 'trade-companion-storage' key. No cloud backup.
Cross-Window Not needed — Single window app. C# process is the single source of truth. BroadcastChannel + Storage events — Main window broadcasts quotes to pop-outs via BroadcastChannel. State synced via storage events.
📈 Subscription & Quote Management
AspectLegacy DesktopNext.js Web
SubL1 Calls Tight loop (zero delay) — QuoteManager.cs iterates watchlist, calls SubL1 per symbol sequentially. Blazing fast because it's server-to-server. Promise.all (parallel) — Browser fires all SubL1 calls concurrently. Fixed from original 150ms throttle.
Stale Detection None — Relies on SignalR auto-reconnect. If connection drops, C# reconnects and re-subscribes. 3-minute timeout — Detects symbols with no update for 3+ min during market hours. Auto re-subscribes stale symbols.
Quote Batching 250ms timer (C#) — QuoteManager collects ticks, broadcasts buffered snapshot every 250ms. 250ms timer (JS) — useSignalR collects ticks in quoteBatchRef, flushes to store every 250ms.
Reconnection C# SigRCoreClient — Exponential backoff. Token refresh every 45 min. Re-subscribes all symbols on reconnect. SignalR withAutomaticReconnect — [0, 2s, 10s, 30s] retry schedule. Re-subscribes on reconnect. Token refresh every 40 min.
🔔 Alert Architecture
AspectLegacy DesktopNext.js Web
Pattern Centralized (OOP) — IAlert interface. Each alert type is a class (PRs.cs, Filings.cs, TradingView.cs, X.cs). All fire AlertTriggered events routed through AlertService. Decentralized (hooks) — Each alert source is an independent React hook. No central router — each hook calls store.addAlert() directly.
Deduplication Per-alert-class logic — Each class tracks its own seen alerts. PRs checks headline text, Filings checks filing ID, etc. Store-level — addAlert() checks symbol + type + first40chars. Single dedup layer for all sources.
Alert Sources 6: TradingView, PRs (news hub), FilteredPR, CatalystScan, Filings, X (Twitter) 7: SignalR alerts, News hub, Filings polling, Tweets polling, Catalyst polling, TradeExchange polling, Price alerts
PR/News SignalR real-time via News Hub + polling fallback (60s). C# PRs.cs handles both paths. SignalR real-time via useNewsHub + useSignalR (BroadcastNews). Also caught by useTweetsPolling as backup.
Audio SpeechService (C#) — System TTS. Can speak alert text aloud. Web Audio API — new Audio('/alert.mp3').play(). Configurable per alert type.

⚖ Architecture Trade-offs

Latency

Legacy has an extra hop (C# relay) but it's on localhost so negligible. Next.js connects directly to Azure — fewer hops over the wire.

Edge: Next.js

Reliability

Legacy's C# process is always-on with robust reconnection. Next.js depends on browser staying open — tab sleep, refresh = lost state.

Edge: Legacy

Scalability

Next.js can run anywhere, multi-window, multi-device. Legacy is locked to one machine, one window.

Edge: Next.js

State Durability

Legacy persists to files + Cosmos DB (survives crashes). Next.js uses localStorage only — clear browser data = gone.

Edge: Legacy

Alert Processing

Legacy's centralized AlertService gives single control point. Next.js hooks are independent — harder to coordinate but simpler to add new sources.

Tie

Deployment

Legacy requires .NET runtime + manual updates. Next.js auto-deploys via Vercel, works in any browser.

Edge: Next.js

Debugging

Legacy has Serilog per-feature log files — excellent for post-mortem. Next.js has only console.log in browser devtools.

Edge: Legacy

Multi-Window

Next.js has full pop-out support with BroadcastChannel sync. Legacy is single-window only.

Edge: Next.js

Code Simplicity

Next.js is ~2K LOC hooks + components. Legacy is 7 C# projects + React. Same features, 3-5x less code.

Edge: Next.js
