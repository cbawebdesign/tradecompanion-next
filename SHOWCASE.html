04 / Detail

What we built.

click to expand
[ A ]
Alert Intelligence
7 features
A.01+

FilteredPR Keyword Matching

Ported regex logic from legacy C# — comma = OR, & = AND, ! = NOT, * = wildcard.

Full regex-based filter system matching the legacy desktop app's behavior. Positive keywords highlight matching PRs in green, negative keywords highlight in red.
  • OR matches: offering,merger,acquisition
  • AND matches: FDA & approval
  • NOT filters: !shelf,!dilution
  • Wildcards: *acquisition*
A.02+

Multi-Symbol Tweet Fix

Legacy only alerted on the first watchlisted cashtag. We fire for every match.

A tweet mentioning $AAPL $TSLA $NVDA now generates three alerts if all three are watchlisted — one per symbol, each with its own dedupKey. Legacy would have surfaced only the first.
A.03+

ExcludeFilings Blacklist

Pipe-separated form types to mute across both SignalR and polling paths.

Noisy filings like Form 4 insider transactions and SC 13G ownership disclosures can be silenced entirely. Config lives in settings: excludeFilings: "SC 13G|4|D"
A.04+

Alert Auditor Safety Net

Polls /api/AlertsBySymbol every 60s. Recovers anything the real-time feed drops.

For each watchlisted symbol the auditor asks the backend: what alerts exist that the web app hasn't logged yet? Any gap gets replayed with its original timestamp. Runs only during market hours.
  • Initial delay 5 minutes — lets hooks settle
  • Dedup checks fresh store state before adding
  • Source tag: useAlertAuditor
A.05+

DedupKey Tagging

Explicit source-derived keys: filing:cik-dcn, tweet:id_long, cat:sym-time.

Every alert carries a unique dedup key inferred from the event source. When SignalR and polling both deliver the same filing, the store recognizes them as the same event via filing:320193-000032019326000012 regardless of message formatting.
  • Filings: filing:<cik>-<dcn>
  • Tweets: tweet:<id_long>
  • Catalysts: cat:<symbol>-<saveTime>
  • PRs: pr:<story_id>
  • TradingView: tv:<alertId>
A.06+

Catalyst PR Linking

resource_id now flows backend → frontend. Catalyst alerts open the actual PR.

Added resource_id to the Catalyst C# model, pushed it through the SignalR broadcast and the AlertsBySymbol endpoint. Frontend constructs /api/pr?id=<resource_id> on click — catalyst alerts now open the originating press release in one click.
A.07+

Airtable Direct API

Replaced broken miniextensions RSS bridge with the native Airtable API.

Old flow: Airtable → miniextensions RSS adapter → web (frequently broken). New flow: Airtable API directly with view-scoped pagination. Three views map to alert types: YouTube, Substack, Articles.
[ B ]
User Experience
8 features
B.01+

Editable Data Ribbon

Double-click any field in the ribbon. Type. Enter. Saved to Cosmos.

Company data — float, sector, market cap, manual overrides — all editable inline. Changes persist to StockData in Cosmos and sync across devices.
B.02+

Five Color Themes

Dark, Blue, Wall Street, Crimson, Nebula — built on CSS variables.

Theme switching via data-theme attribute on documentElement. Full color palette swap — backgrounds, borders, accents, alert colors — all reactive to user preference stored in the config.
B.03+

Pop-Out Windows

AlertBar, Watchlist, Scanner, Mascot — each can detach into its own window.

Multi-monitor workflows supported natively. Pop-outs communicate via BroadcastChannel for live quote updates and localStorage events for state sync.
B.04+

Grok AI Summaries

One-click summarization of PRs and filings via Grok.

A Grok button appears on any alert with a URL. Click it — Grok fetches the page, summarizes the relevant content, surfaces key financial signals. Much faster than reading the full filing.
B.05+

Text-to-Speech

Web Speech API reads alerts aloud. 30-second dedup on identical phrases.

Never miss a PR while you're looking away from the monitor. Voice selection, rate, and volume are user-configurable. Dedup prevents repeated identical phrases within a 30-second window.
B.06+

Per-Type Alert Sounds

Frequency and duration configurable per alert type.

Distinct audio signatures per event type — filings sound different from tweets sound different from price triggers. Configured in Settings → Audio & Speech.
B.07+

Alert Mascot

Animated character that reacts to incoming alerts. Tasteful levity.

Small draggable mascot lives in a pop-out window. Subtle animations trigger on alert arrival. Can be disabled for minimalist workflows. Because trading desks can be fun.
B.08+

Clickable URLs

Every alert type — tweets, PRs, filings, catalysts — links to its source.

Tweet URLs built from x.com/<user>/status/<id_long>. Filing URLs from EDGAR. PR URLs from resource_id. Catalysts link to the originating press release. Every alert is one click from its source.
[ C ]
Infrastructure
6 features
C.01+

Cosmos Watchlist Sync

Watchlists survive browser clear, work across devices, debounced save.

useCosmosSync hook reconciles local Zustand state with Cosmos DB user documents. 3-second debounce on writes. Pull on mount. Push on change.
C.02+

Cross-Window Sync

BroadcastChannel for quote updates, storage events for state sync.

Open the Watchlist in a pop-out and flag a symbol — the main window reflects it instantly. Same for quotes, alerts, theme changes. Zero network round-trips.
C.03+

AHK Companion Server

Local Node.js bridge so browser-side clicks can fire AutoHotKey scripts.

20-line companion Node server listens on localhost:9876. The web app posts a symbol — the server invokes an AHK script with that symbol as an argument. Unlocks all the legacy desktop keyboard/automation workflows inside a web context.
C.04+

Alert Integrity Agent

Azure Function timer (5m). Reconciles client log vs. backend truth.

Every 5 minutes during market hours, a timer-triggered Azure Function compares what the web client logged to what the backend actually dispatched. Mismatches flag as missed / delayed / duplicate. Admin dashboard surfaces the daily score.
C.05+

TC Context Layer

6 knowledge modules in Cosmos: source map, taxonomy, components, config, baselines, issue log.

Persistent structured knowledge the agents and auditors read from. Each module is a versioned Cosmos document describing one aspect of the system — data sources, alert taxonomy, UI component map, live config, performance baselines, known issue history.
C.06+

Settings UI

Every config field from the legacy app, plus all the new ones, surfaced in a sane UI.

TradingView ID, API keys, excludeFilings, filteredPrPositive/Negative, showAllTradeExchange, per-type sounds, AHK URL, theme, TTS voice/rate — all user-editable. No more editing JSON files.
